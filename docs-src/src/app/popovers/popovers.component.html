Popovers

Bootstrap popovers based on data- attributes and (optionally) ng-template's.

API

NzbPopoverDirective NzbPopoverDirective on GitHub extends NzbAbstractPopup NzbAbstractPopup on GitHub

Options

  • title
    • Use the title or data-title attributes (example.)
    • Use [nzbPopoverTitle]="myTitleTemplate" to specify a template (example.)
  • content
    • Use data-content.
    • Use [nzbPopoverContent]="myContentTemplate" to specify a template (example.)
  • placement
    • Use data-title.
    • Use [nzbPopoverPlacement]="myPlacementFunc" to specify a function (example.)
  • All the other Bootstrap popover options (except selector) are supported via data- attributes (example.)
    • data-animation
    • data-container
    • data-delay
    • data-html
    • data-trigger
    • data-offset
    • data-fallbackTrigger

Methods

The directive wraps all Bootstrap popover methods except for dispose() (the directive takes care of disposing of the popover.)

  • show()
  • hide()
  • toggle() show/hide/toggle example
  • enable()
  • disable()
  • toggleEnabled() enable/disable/toggleEnabled example
  • update()

The directive also exposes the following public API:

  • Observables (example)
    • getEvents(): Observable<Event> An observable of the native popover events.
    • getStatus(): Observable<string> An observable of 'show'|'inserted'|'shown'|'hide'|'hidden'.
  • getPopupData(): any Returns the data from the underlying Bootstrap component. (example)

Examples

Using attributes for title and content

The directive can handle the simplest use case (e.g. adding a popover with a static title and content) using nzbPopover title="..." data-content="...".
