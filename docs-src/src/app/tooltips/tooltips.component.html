Tooltips

Bootstrap tooltips based on data- attributes and (optionally) an ng-template.

API

NzbTooltipDirective NzbTooltipDirective on GitHub extends NzbAbstractPopup NzbAbstractPopup on GitHub

Options

  • title
    • Use the title or data-title attributes (example.)
    • Use [nzbTooltipTitle]="myTemplate" to specify a template (example.)
  • placement
    • Use data-title.
    • Use [nzbTooltipPlacement]="myPlacementFunc" to specify a function (example.)
  • All the other Bootstrap tooltip options (except selector) are supported via data- attributes (example.)
    • data-animation
    • data-container
    • data-delay
    • data-html
    • data-trigger
    • data-offset
    • data-fallbackTrigger

Methods

The directive wraps all Bootstrap tooltip methods except for dispose() (the directive takes care of disposing of the tooltip.)

  • show()
  • hide()
  • toggle() show/hide/toggle example
  • enable()
  • disable()
  • toggleEnabled() enable/disable/toggleEnabled example
  • update()

The directive also exposes the following public API:

  • Observables (example)
    • getEvents(): Observable<Event> An observable of the native tooltip events.
    • getStatus(): Observable<string> An observable of 'show'|'inserted'|'shown'|'hide'|'hidden'.
  • getPopupData(): any Returns the data from the underlying Bootstrap component. (example)

Examples

Example: Use the title attribute

The directive can handle the simplest use case (e.g. adding a tooltip with a static title) using nzbTooltip title="...".

Example: Use nzbTooltipTitle

The directive can also handle more complex situations, by using ng-template to define the tooltip's content (i.e. with variable interpolation,) and by wrapping the Bootstrap tooltip API and events.

Use [nzbTooltipTitle]="myFancyTip" in the target element to use the template at <ng-template #myFancyTip>...</ng-template> as the tooltip. You can interpolate and *ngIf to your heart's content within the template.

A couple of notes about using templates:

  • The title attribute, if present, will always override [nzbTooltipTitle]. This is intentional. Don't use both in the same element.
  • If you use a template, the tooltip's html property will be set to true. This is intentional — Angular needs markup to do its stuff. If you don't need to do logic or interpolate values in the tooltip, just use the title attribute or a function.

Example: Using a function for placement

Use [nzbTooltipPlacement]="myFunc" if you want to do some fancy calculation to place the tooltip.
