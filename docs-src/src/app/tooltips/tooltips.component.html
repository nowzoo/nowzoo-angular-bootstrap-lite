Tooltips

Bootstrap tooltips based on data- attributes and (optionally) an ng-template.

API

NzbTooltipDirective NzbTooltipDirective on GitHub extends NzbAbstractPopup NzbAbstractPopup on GitHub

Selector: [nzbTooltip]

Inputs

  • nzbTooltipTitle: TemplateRef example Or use the title or data-title attribute. example
  • nzbTooltipPlacement: (tooltipEl, targetEl) => string Or use the data-placement attribute.

All the other Bootstrap tooltip options (except selector) are supported via data- attributes:

  • data-animation
  • data-container
  • data-delay
  • data-html
  • data-trigger
  • data-offset
  • data-fallbackTrigger

Methods

The directive wraps all Bootstrap tooltip methods except for dispose() (the directive takes care of disposing of the tooltip.)

  • show()
  • hide()
  • toggle()
  • enable()
  • disable()
  • toggleEnabled()
  • update()

Properties

  • shown: boolean Whether the tooltip is completely shown (i.e., when the animations are done.)
  • hidden: boolean Whether the tooltip is completely hidden (i.e., when the animations are done.)
  • enabled: boolean Whether the tooltip is enabled.
  • events: Observable<Event> An observable of the native tooltip events.
  • status: Observable<string> An observable of 'show'|'inserted'|'shown'|'hide'|'hidden'.
  • boostrapComponentData: any The underlying tooltip data attached by Bootstrap to the target element.

Using an attribute for the title

The directive can handle the simplest use case (e.g. adding a tooltip with a static title) using nzbTooltip title="...".

You can use data-title rather than title.

Using a template the for title

The directive can also handle more complex situations, by using an ng-template to define the tooltip's title. This allows you to interpolate variables and perform more complex logic.

This demo displays an informative message about how long it's been since you toggled the tooltip.

Use [nzbTooltipTitle]="titleTemplate" with <ng-template #titleTemplate>...</ng-template>

The title attribute, if present, will override [nzbTooltipTitle]. This is by design. If you're using a template for the title, omit the title attribute.

If you use a template for the title, the tooltip's html property will be set to true. Angular needs markup to do its stuff.

Using a function for placement

If you want to do some fancy calculation in order to figure out placement, use [nzbTooltipPlacement]="myPlacementFunc".

As an alternative, try plain old data-placement="auto". This should be sufficient in most use cases.

Our function in this example returns 'top' if the window is smaller than the small breakpoint, and 'left' otherwise. But by default Bootstrap tries to flip the placement if there isn't enough room in the viewport to display the entire tooltip — so your results may vary.

Getting the tooltip directive instance

Access the tooltip directive instance in a template variable with #myTooltip="nzbTooltip". You can then use myTooltip in your template, or use @ViewChild('myTooltip') myTooltip: NzbTooltipDirective to manipulate it in typescript.

This example toggles an extra .danger class on the tooltip, using theboostrapComponentData property. This gives us access to tip property, which is the top level HTML element for the tooltip.
