Method

handleUpdate(): void

A wrapper around the native Bootstrap $().modal('handleUpdate') method.

Use this when the height of the modal changes. It ensures that scrollbars do not suddenly appear when the modal's height exceeds the height of the viewport.

Property

events: Observable<Event>

An observable of the modal's native Bootstrap events.

  • Example
  • Typescript
  • HTML

Property

status: Observable<string>

An observable of a string representing the modal's status. The value is:

  • 'unshown' when the modal does not exist in the DOM — i.e. before it has been shown or after it's been hidden.
  • 'show' while the modal is being animated to shown — i.e. after the show.bs.modal event.
  • 'shown' while the modal is shown — i.e. after the shown.bs.modal event.
  • 'hide' while the modal is being animated to hidden — i.e. after the hide.bs.modal event.
  • 'hidden' after the hidden.bs.modal event. Note that immediately afterward the directive disposes of the modal markup and Angular bindings, and sets the status back to 'unshown'.

This example uses modalInstance.status | async to display the modal's current status.

Options

In keeping with our simple-is-best, non-invasive philosophy, the directive does not have options (i.e. inputs) for things like animation or backdrop. Instead, you set these options the native way, via data- attributes or CSS classes.

You can't set data-show="true". To show the modal immediately (say, on navigating to a new route,) call modalInstance.show() programatically in the component that contains the modal. See this example. All the other Bootstrap modal data- attributes are supported.

Backdrop

By default the backdrop is shown, and clicking on it will dismiss the modal.

Set the data-backdrop attribute in the top level .modal tag to change this.

Possible values are data-backdrop="false", for no backdrop, and data-backdrop="static", for a backdrop that does not dismiss the modal.

  • Example
  • HTML (full)

Use data-backdrop="false" or data-backdrop="static".

Keyboard

By default, pressing the esc key will dismiss the modal when it is focused.

Use data-keyboard="false" in the top level .modal tag to disable this behavior.

  • Example
  • HTML (full)

The first example sets data-keyboard="false".

The second example (default) omits the data-keyboard attribute. This is the default behavior.

Focus

By default, the modal is focused when shown.

Use data-focus="false" in the top level .modal tag to disable this behavior.

Note that you must also set tabindex="-1" in the .modal tag for the modal to receive focus.
